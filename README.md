## Waterfall Methodology Breakdown for SmartEco

### 1. Requirements Analysis
* **Objective:** Define what the product must do.
* **Activities:** Gather hardware constraints (IoT sensors), user interface requirements (Mobile App), and eco-regulatory compliance standards.
* **Deliverable:** Software Requirements Specification (SRS) document.

### 2. System Design
* **Objective:** Architect both hardware and software elements.
* **Activities:** Sketch UI/UX wireframes, map cloud database schemas, and choose communication protocols (Zigbee/Wi-Fi).
* **Deliverable:** Technical Design Document (TDD) and System Architecture Diagram.

### 3. Implementation (Coding/Building)
* **Objective:** Develop the actual product modules.
* **Activities:** Write backend logic in Python/Node.js, build iOS/Android apps, and program microcontrollers for smart plugs.
* **Deliverable:** Source code uploaded to staging branches and functional hardware prototypes.

### 4. Integration & Testing
* **Objective:** Ensure the software works seamlessly with the hardware safely.
* **Activities:** Perform Unit Testing on APIs, Integration Testing between app and sensors, and User Acceptance Testing (UAT).
* **Deliverable:** Test cases log and QA Sign-off report.

### 5. Deployment
* **Objective:** Launch the final product to the consumer market.
* **Activities:** Publish mobile apps to Google Play/App Store, deploy backend services to AWS cloud, and ship initial hardware batches.
* **Deliverable:** Live production application and user onboarding manual.

### 6. Maintenance
* **Objective:** Patch bugs and provide continuous updates.
* **Activities:** Monitor server uptime, roll out over-the-air (OTA) firmware updates, and manage customer support tickets.
* **Deliverable:** Post-launch patch logs and performance optimization metrics.
